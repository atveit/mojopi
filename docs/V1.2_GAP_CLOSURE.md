# v1.2 Gap Closure — functional parity with pi-mono

v1.1 shipped four beyond-the-port features but the core pi-mono port has
concrete **functional** gaps. UI polish (streaming display, TUI auto-launch,
doctor command) is deferred to v1.3. This document plans closure of the
must-have functional gaps only.

## Functional gaps being closed

| # | Gap | Why it's functional |
|---|-----|---------------------|
| 1 | `--session <id>` parses but doesn't rehydrate history | Resume is advertised but broken |
| 2 | Memory retrieval exists but isn't auto-injected into the loop | Retrieval happens only if called manually; auto-recall is the value prop |
| 3 | Reasoning-model `<think>` blocks leak into output and pollute extracted tool calls | Any reasoning-class model (DeepSeek-R1, QwQ, Qwen3-Thinking) is broken without this |
| 4 | `MAX_PARSE_RETRIES = 3` is declared but never used | Malformed tool-call JSON kills the turn instead of being re-requested |
| 5 | Compaction module exists but isn't wired into `loop.mojo` | Long sessions blow context without automatic trim |
| 6 | Turn-cap hit returns placeholder `"[max tool iterations reached]"` | Should summarize state so the user can resume |
| 7 | Tool-calling never empirically verified against a tool-capable model | ReAct loop works on Qwen (chat model, hallucinates tools) — Llama-3.1-Instruct path is untested |

## Deferred to v1.3 (UI / polish, not functional)

- TUI auto-launch + token streaming display (textual app is built and tested; just not wired)
- Extra slash commands (`/model`, `/history`, `/save`, `/fork`)
- `doctor` and `models` CLI subcommands
- Linux + CUDA NFR validation (no hardware)
- Conda channel publishing

## Design — 6 parallel agents

Each agent owns a standalone Python module + tests. **No agent modifies
`main.mojo` or `loop.mojo`**; lead wires everything at the end via the
functional APIs the agents expose. This pattern is confirmed across W3,
R1, R2, R3, and v1.1 runs — zero merge conflicts every time.

| Agent | Owns | Exposes |
|-------|------|---------|
| **A** Session resume | `src/agent/session_manager.py`, `session_manager.mojo`, `tests/test_session_manager.py` | `load_session_history(id_or_prefix) -> list[HistoryEntry]`, `save_turn(session_id, entry)`, `new_session_id()` |
| **B** Auto-memory injection | `src/coding_agent/memory/auto_inject.py`, `tests/test_memory_auto_inject.py` | `augment_system_prompt(sp, query, k=3) -> str`, `extract_after_session(session_id)` |
| **C** Thinking-token handling + parse retry | `src/agent/thinking.py`, `src/agent/parse_retry.py`, `tests/test_thinking.py`, `tests/test_parse_retry.py` | `strip_thinking(text) -> (visible, thinking)`, `retry_parse_tool_calls(text, regenerate_fn, max_retries=3)` |
| **D** Compaction + turn-cap summary | `src/agent/compaction_bridge.py`, `src/agent/turn_summary.py`, `tests/test_compaction_bridge.py`, `tests/test_turn_summary.py` | `auto_compact_if_needed(history, tokens) -> list[HistoryEntry]`, `summarize_turn_cap(history) -> str` |
| **E** Session ID resolution | `src/agent/session_resolver.py`, `session_resolver.mojo`, `tests/test_session_resolver.py` | `resolve_session_id(prefix) -> str`, `list_all_sessions() -> list[dict]`, `get_latest_session_id() -> str` |
| **F** Tool-calling empirical | `scripts/verify_tool_calling.py`, `tests/test_real_tool_calling.py` | `verify_tool_calling(model_repo, tool) -> dict`; empirical test marked `@pytest.mark.slow` |

### Test requirements

- Each agent delivers **6+ unit tests** using monkeypatching / mocks.
- Agent F's empirical test gates on model availability and skips cleanly.
- Target: **+40 tests**, total **~305 unit + ~12 empirical**.

## Lead integration (after all 6 agents complete)

### `main.mojo` changes (small)

- When `--session <prefix>` is passed: call Agent A `load_session_history` +
  Agent E `resolve_session_id` to get full ID → replay into `AgentContext`.
- When a turn completes: call Agent A `save_turn(session_id, entry)`.
- REPL `/exit`: call Agent B `extract_after_session(session_id)` as a
  fire-and-forget.

### `loop.mojo` changes (small)

- Before each generate call: call Agent B `augment_system_prompt(sp, last_user_msg)`
  when `--with-memory` is set (or always, via env var).
- After generate: call Agent C `strip_thinking(response_text)` before extracting
  tool calls.
- If `extract_tool_calls` returns empty AND response contains `{"name":` JSON
  fragments: call Agent C `retry_parse_tool_calls` up to 3×.
- Before format_history_as_chatml: call Agent D `auto_compact_if_needed` when
  cumulative token estimate > 75% of context.
- If `MAX_TOOL_ITERATIONS` hit: call Agent D `summarize_turn_cap(history)`
  instead of returning `"[agent: max tool iterations reached]"`.

### Verification

1. `pixi run test` — all 305+ unit tests pass.
2. `pixi run bash -c "PYTHONPATH=src pytest tests/test_real_tool_calling.py -v -m slow"` —
   at least one empirical path passes against a cached tool-capable model.
3. Manual REPL smoke test: launch, `--session <latest>`, verify history replay.
